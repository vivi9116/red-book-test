# 经验记录

- 免费测试轮播图不能用“空背景 + 后期硬塞正文”的方式做手帐风；如果用户要“错字那版”的效果，必须先让底图拥有明确的大块题卡/日记页承载区，再叠准确文字。
- 豆包直接生成中文长文容易出错；正文必须程序叠加，但版式要像原图那样为文字设计，而不是让文字去躲装饰。
- 当用户明确认可“错字那版”的整体质感时，不要重构排版或重画题卡；应优先复用同款提示词重新生成，或者只做无痕修复。明显补丁会破坏可发布质感。
- 用户说“是这版”时，指的是豆包原生手帐拼贴图：撕纸标题、翻开日记本、照片贴纸、人物剪贴、文字融合在纸面里。后期卡片化、白框化、规整化都会偏离目标。
- 图片模型只要直接写长中文，就会随机出现错字；可发布方案必须让模型生成无文字手帐底图，再用确定性文字渲染写到纸面位置。
- 底部提示文案要尽量短，避开容易被看错或写坏的字，比如“答题分数”不如“记下分数，最后算总分”稳。
- 免费测试新选题要轻，适合评论区报分；比起深度解释，更适合做“偏爱 vs 理解”“黏人 vs 独立”这种一眼能代入的二选一倾向。
- 同风格不等于同排版；新的免费测试要换构图隐喻，比如题卡拼贴、关系线索板、聊天票据，而不是继续套上一套笔记本模板。
- 文字要融合进图片，必须先让底图有承载文字的题卡、编号章、选项标签和撕纸条；不能把正文直接压在空背景或日记本上。
- 如果用户给的参考是“一整张纸面直接写题”，不要再拆成多张漂浮题卡；颜色也要按测试主题重新定，不能沿用上一套豆沙红/奶油黄。
- 用户明确说“目前的不能用、不要在原来上修改”时，必须停止局部修补，直接重做生成结构；否则只是在错误方案上越改越乱。
- 程序确定性画出来的手帐图容易变成干净模板，达不到用户要的“豆包原生精致拼贴感”；正确路线是豆包重新生成无字精致底图，程序只负责准确文字。
- 当用户明确说“文字还是不能完全融合到版面”时，不能继续程序叠字；要按用户要求让豆包把图片和文字一起生成，优先解决整体融合感，再人工检查错字。
- 付费测试新增产品时必须只新增独立 `test id`、独立内容和独立图片目录；不能替换旧测试配置、旧入口或旧图片，发布前要同时检查旧测试链接仍能打开。
- 用户说“重新做测试网站”时，默认是重做整个客户测试页面：首屏、解锁区、答题区、报告区都要统一，不是只换首屏图片或调一处卡片。
- 用户说“按之前的排版来，只是 UI 精美一点”时，不能再改信息结构和页面顺序；只允许在原排版上优化质感、间距、颜色、图片融合和移动端细节。
- 付费测试预览图不能把素材空白区大面积裁出来；如果图片本身有留白，要用放大、偏移和柔和遮罩把视觉重点落在人物、物件和主题线索上。
- 手机端不要用大文字纸片盖住核心插图；小屏应优先改成上文案下图片，保证底图能被看见。
- 付费测试首屏预览不能只用线条、框和示意图凑版面；用户会把它理解成未完成。应优先使用该测试自己的图片资产做精致拼贴，再用少量标签和便签解释报告价值。
- 当用户说“做测试网站”时，不要只换首屏图；要检查入口、兑换、答题、报告整套体验是否统一，尤其手机端不能出现大块空白、硬拼图片或文字挤压。
